# Difficulty Curve Design Guide

## The Ideal Curve

```
Difficulty
  ^
  |                                    /
  |                               ___/
  |                          ___/
  |                     ___/
  |                ___/
  |           ___/
  |      ___/
  | ___/
  |/
  +----------------------------------------> Progress
    Tutorial  Early   Mid    Late   End
```

## Phase Breakdown

### Tutorial Phase (Chapter 1)
- All tasks trivially easy
- Teach basic mechanics: open quest book, submit item, claim reward
- Use checkmark tasks for "read the book" type quests
- Rewards should be generous to build confidence

### Early Phase (Chapter 2-3)
- Introduce resource gathering and basic crafting
- Tasks: collect 16-64 of common items, craft basic tools
- Introduce optional side tasks for extra rewards
- First bottleneck task should be achievable in < 30 min

### Mid Phase (Chapter 4-6)
- Multi-step processing chains
- Tasks: craft machine parts, process ores, build structures
- Introduce exploration tasks (location, advancement)
- Bottleneck tasks may take 1-2 hours
- Reward economy should support automation investments

### Late Phase (Chapter 7-8)
- Complex automation and multi-block structures
- Tasks: build reactor, automate crafting chain, defeat boss
- Kill tasks with challenging mobs
- Bottleneck tasks may take several sessions
- Rewards should be spectacular

### Endgame (Final Chapter)
- Creative-adjacent challenges
- Tasks: defeat ender dragon, build mega-factory, collect all items
- Milestone rewards should be memorable

## Difficulty Indicators to Watch
- **Task quantity per chapter**: 10-15 (tutorial), 15-25 (early), 20-35 (mid), 25-40 (late)
- **Item stack sizes**: 1-16 (tutorial), 16-64 (early), 64-256 (mid), 256+ (late)
- **Mob difficulty**: Passive mobs → Overworld hostile → Nether → End → Custom bosses
- **Exploration distance**: Spawn area → 100 blocks → 1000 blocks → Other dimensions

## Bottleneck Task Design
- Should be challenging but NOT frustrating
- Always provide alternative paths or hints in description
- Use description to explain WHY this is hard and what skills it tests
- Reward bottleneck tasks significantly more than regular tasks
- Mark them as milestone tasks (pentagon shape, gold color)
